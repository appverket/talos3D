# Support

Talos3D is maintained on a time-permits basis.

## What To Expect

- There is no support SLA.
- There is no guarantee of a reply to issues, questions, or pull requests.
- Sponsorship or donations do not purchase support, priority, or roadmap
  influence.
- Commercial use of the code does not create any support entitlement.

## Where To Ask

- Use GitHub issues for reproducible bugs and focused feature proposals.
- Use pull requests for concrete fixes that fit the project architecture.

Before filing an issue, check the repository docs and existing issues first.

## Good Requests

- minimal reproduction cases
- precise bug reports
- documentation fixes
- changes that reduce maintenance burden

## Poor Requests

- vague "please implement X" requests with no design
- demands for timelines or guaranteed responses
- private support expectations without a separate written agreement

Security issues should follow [SECURITY.md](./SECURITY.md), not public bug
reporting.
